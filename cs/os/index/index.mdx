어셈블리어와 같은 언어의 등장으로 프로그램을 작성하는 데 일정한 패턴과 기능이 반복되는 것을 발견했다.
폰 노이만은 이를 하드웨어와 소프트웨어로 분리하는 내장 프로그램 컴퓨터 개념을 최초로 발표했다.
공통 작업을 계속 반복해서 프로그램마다 구현하는 것은 비효율적이었기에, 공통 작업을 하나의 프로그램으로 묶어 다른 프로그램들이 공유할 수 있도록 했다.
공통 작업에서 <mark>CPU, 메모리, I/O 장치와 같은 하드웨어 자원과 상호작용을 다루는 작업은 복잡하므로 이를 대신해주는 소프트웨어 OS</mark>가 등장했다.

# Kernel

커널은 자원이 필요한 프로세스에 자원을 나눠주고, 프로세스를 작업 관리 및 제어하고, 메모리를 제어하는 등 모든 핵심 작업을 수행한다.
따라서 운영체제의 가장 아래 계층에서 들어가고, 커널 코드는 구체적으로 함수의 형태로 존재하여 응용프로그램이 커널에 있는 함수를 활용하기 위해 시스템 콜을 사용한다.
즉 커널은 스스로 실행되는 프로세스가 아닌 호출되어 실행되는 함수, 데이터의 집합이다.
